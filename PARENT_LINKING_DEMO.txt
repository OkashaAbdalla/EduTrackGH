  children: []  ← EMPTY (no children yet)
}

═══════════════════════════════════════════════════════════════
STEP 2: TEACHER PROPOSES STUDENT
═══════════════════════════════════════════════════════════════

Teacher logs in → "Manage Students" → "Propose Student"

Fills form:
┌─────────────────────────────────────┐
│ Student ID: P1-2026-001             │
│ Full Name: Ama Mensah               │
│ Gender: Female                      │
│ Classroom: Primary 1A               │
│                                     │
│ Parent Information:                 │
│ Parent Name: Mrs. Mensah            │
│ Parent Email: mensah@gmail.com  ←── MATCHES!
│ Parent Phone: 0241234567            │
└─────────────────────────────────────┘

System creates:
{
  studentId: "P1-2026-001",
  fullName: "Ama Mensah",
  parentEmail: "mensah@gmail.com",
  status: "PENDING",  ← NOT ACTIVE YET
  isActive: false
}

Parent.children = []  ← STILL EMPTY (not linked yet)

═══════════════════════════════════════════════════════════════
STEP 3: HEADTEACHER APPROVES (MAGIC HAPPENS HERE!)
═══════════════════════════════════════════════════════════════

Headteacher logs in → "Student Management" → Sees pending list

┌─────────────────────────────────────────────────────────────┐
│ PENDING STUDENTS                                            │
├─────────────────────────────────────────────────────────────┤
│ Ama Mensah (P1-2026-001)                                   │
│ Class: Primary 1A                                           │
│ Parent: Mrs. Mensah (mensah@gmail.com)                     │
│ Proposed by: Teacher Grace                                 │
│                                                             │
│ [Reject]  [Approve] ← Clicks Approve                       │
└─────────────────────────────────────────────────────────────┘

SYSTEM EXECUTES (Automatically):

1. Update Student:
   student.status = "ACTIVE"
   student.isActive = true
   student.approvedBy = headteacher._id
   student.approvedAt = "2026-02-27..."

2. Search for Parent:
   Query: Find User where:
   - role = "parent" AND
   - (email = "mensah@gmail.com" OR phone = "0241234567")
   
   Result: FOUND! ✅

3. Link Student to Parent:
   parent.children.push(student._id)
   parent.save()

RESULT:
Parent Account NOW has:
{
  email: "mensah@gmail.com",
  children: [ObjectId("ama_id")]  ← AMA IS LINKED!
}

WHAT IF PARENT ACCOUNT DOESN'T EXIST?
═══════════════════════════════════════════════════════════════

Teacher proposes: parentEmail = "newparent@gmail.com"
Headteacher approves

System searches: No parent found with that email ❌

Result:
- Student is STILL approved (status: ACTIVE)
- Student CAN be marked for attendance
- Parent info is STORED with student
- NO automatic account creation (security)

Parent must:
1. Register manually at /register
2. Use SAME email: "newparent@gmail.com"
3. Next student approval will link automatically

Alternative (Future):
- System can auto-create parent account
- Send welcome email with temp password
- Requires email verification
